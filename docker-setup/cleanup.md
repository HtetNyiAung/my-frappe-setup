# Cleanup Script (`cleanup.sh`)

The `cleanup.sh` script is a powerful utility used to reset your project environment to a completely fresh state.

## ⚠️ DANGER ZONE

Running this script is **irreversible**. It will:
-   **Stop and Delete** all project containers.
-   **Delete all Data Volumes**: This permanently wipes your MariaDB (Frappe) and PostgreSQL (Keycloak) databases, as well as all uploaded files.
-   **Delete Custom Images**: Removes the local Docker image generated by `setup.sh`.
-   **Prune Build Cache**: Clears orphaned Docker build layers to free up disk space.

## What it does

1.  **Service Teardown**: Runs `docker compose down -v` to remove services and volumes.
2.  **Targeted Deletion**: Only targets resources tagged with your specific `STACK_ID` to prevent accidental deletion of other Docker projects on the same machine.
3.  **Image Removal**: Finds and deletes the local version of your `CUSTOM_IMAGE`.
4.  **Network Pruning**: Removes the virtual networks used by the stack.

## Usage

```bash
chmod +x cleanup.sh
./cleanup.sh
```

## When to use this?

-   When you want to perform a **total system reset**.
-   When you need to **completely rebuild** the stack with fresh database schemas.
-   When decommissioning the server.
-   When you are experiencing deep configuration issues that are easier to fix by starting from zero.
